## Rule
The Node app runs correctly on port 8080 (internal health check passes), but the web server proxy (nginx or alternative) is NOT configured in `/etc/nginx/`. The CD workflow's nginx fix step never fires.

**Why:** `grep -rl "tbjvisionconnect.com" /etc/nginx/` returns nothing; `find /etc/nginx /home/*/conf -name "*.conf"` also returns nothing. Hostinger VPS may use OpenLiteSpeed, Nginx at a non-standard path, or a panel-managed config.

**How to apply:** The next CD run (#472+) includes `nginx -T` dump, broad `grep -rl` across `/etc /usr/local /home`, process listing, and port scan. Read those logs to identify the actual web server and config location. Then update the workflow to target the right path or add a server-specific config.

**Evidence from run #478 logs:**
- L493: `✅ Server is up on port 8080` — app healthy internally
- L582: `Config file: NOT FOUND` — nginx search found nothing
- L583: `=== ALL NGINX CONFIGS ===` + L584 immediately = find also found nothing
- Domain curl returns 000 (SSL mismatch when hitting raw IP, or 502 via domain)
